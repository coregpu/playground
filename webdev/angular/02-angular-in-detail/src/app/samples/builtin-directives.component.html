Using the ngFor directive

The following illustrates the ngFor directive along with all of its template variables (index, odd/even, first/last). Additionally, TrackBy input is also used which is invoked by Angular to get an unique identifier for objects that ngFor iterates over (Product in this case). This can significantly reduce amount of DOM manipulation in cases where the iterable is often updated by recreating its objects, but the underlying data has not changed (hence unique identifier remains the same). Without custom TrackByFunction, ngFor will use object identity as the key which can result in many unneeded DOM updates. Check more at: https://angular.io/api/common/NgForOf
