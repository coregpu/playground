A custom ngFor directive (see for.directive.ts for in-depth description)

The following illustrates the custom appForDirOf directive which mimics the behavior of the built-in ngFor directive. The radio buttons right under this paragraph allow to target different tables with the Product changes. The first table doesn't specify the TrackBy function for uniquely identifying objects, while to other tables do. Try adding, removing, shifting, recreating products with browser console opened to see the changes processed by the directive. Also note that there are fewer changes when custom TrackBy function is provided ( meaning performance is better). See for.directive.ts for in-depth description of how appForDirOf works.

The button under the table starts or stops the timer which periodically updates the price of each product in the repository. Note that this change is automatically picked up by Angular and relevant bindings are re-evaluated to reflect the changes (in other words, appForDirOf directive is not involved in this). This is done as part of change detection process, which is triggered each time the timer fires at which point Angular re-evaluates the bindings and picks up on changes in prices.
