Illustrates passing a style group to the animate function

The following illustrates that it's possible to pass a style group as the second argument to the `animate` function. This has an effect of adding a new state between the to states specified in the transition, in that the CSS values first change from the initial state to the values specified in this style group, and than they change to the final state. Check the `animateCallWithStyleGroup` trigger defined in animation.component.ts.

Illustrates an array of animations to the `transition` function

The following illustrates passing an array of animations to the `transition` function. Angular will execute all of these in sequence, and as long as the last specified animation doesn't define the style group, the final animation will transition to the style specified in the target state. Check the `transitionWithArrayOfAnimations` trigger defined in animation.component.ts.

Illustrates parallel animations

The following illustrates using the `group` function to create animations that are run in parallel by Angular. All the animations will start at the same time however may have different durations. Check the `parallelAnimations` trigger defined in animation.component.ts.
